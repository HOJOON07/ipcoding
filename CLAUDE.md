# CLAUDE.md — 입코딩 (IpCoding)

터미널 AI 에이전트 코딩용 macOS 로컬 음성 입력 앱. Swift 메뉴바 앱 + whisper.cpp(STT) + llama.cpp(교정·정돈 LLM).

## 문서 체계 — 코드보다 문서가 상위다

| 문서 | 역할 | 경로 |
|---|---|---|
| PRD | 무엇을, 왜 | `docs/PRD.md` |
| 기술설계서 (TDD) | 어떻게 — 모듈 명세, 상태 머신 | `docs/TDD.md` |
| 구현계획서 | 어떤 순서로 — 태스크 ID, 완료 기준 | `docs/PLAN.md` |
| Phase 0 벤치 명세 | 벤치마크 절차 | `docs/BENCH.md` |

**철칙: 구현과 문서가 어긋나면 코드가 아니라 문서를 먼저 판정한다.** 설계 변경이 필요하면 (1) `spec-guardian` 에이전트로 충돌 지점 확인 → (2) 문서 수정 제안을 사용자에게 승인받고 → (3) 문서 갱신 → (4) 코드 작업. 문서에 없는 동작을 임의로 구현하지 않는다.

모든 작업은 구현계획서의 태스크 ID(예: `1.7`)로 지칭한다. 태스크 착수 전에 TDD의 해당 모듈 섹션을 반드시 읽는다.

## 아키텍처 불변 규칙

- 모듈 구조는 TDD §1을 따른다. 모듈 간 의존은 단방향: 모든 모듈 → `SessionCoordinator`로 이벤트 전달, 역방향 직접 호출 금지.
- **상태 전이는 `SessionCoordinator`만 수행한다.** 다른 모듈이 상태를 바꾸거나 자체 상태 머신을 갖지 않는다. 전이표는 TDD §2가 유일한 정본.
- UI 갱신과 상태 머신은 `@MainActor`. 추론(whisper/llama)은 전용 백그라운드 큐, 결과만 MainActor로.
- HUD(NSPanel)는 절대 key window가 되지 않는다 (포커스를 뺏으면 주입이 무너진다).
- LLM 실패 시 항상 원문 폴백 (PRD 원칙 3). 폴백 경로를 제거하는 변경은 금지.

## 코딩 규칙

- Swift 5.9+, 강제 언래핑(`!`) 금지(테스트 제외), 에러는 typed throw 또는 Result로 코디네이터까지 전파.
- 새 의존성 추가는 사용자 승인 필요. 현재 허용: whisper.cpp, llama.cpp, Sparkle(Phase 3).
- 개인 데이터(음성 원본, 전사·교정 결과 텍스트)를 디스크에 쓰거나 로그에 평문으로 남기지 않는다. 음성 버퍼는 메모리에만 두고 세션 종료 시 폐기한다. os.Logger는 privacy 마스킹. **예외**: 개발 검증용 임시 덤프는 `#if DEBUG` 한정·고정 파일명 덮어쓰기·내용 미로깅(경로만) 조건에서만 허용하며, 명시된 제거 태스크(PLAN)를 완료 기준에 건다.
- 커밋: 태스크 단위, 메시지는 `[1.7] PasteboardInjector: 백업/복원 구현` 형식. 모델 파일(*.bin, *.gguf)과 `~/Library` 산출물은 커밋 금지(.gitignore 확인).

## 검증 루프 — "빌드가 되면 끝"이 아니다

1. 빌드/테스트 실행은 직접 하지 말고 **`build-runner` 에이전트에 위임**한다 (로그가 메인 컨텍스트를 오염시키지 않도록).
2. 태스크 완료 전 `code-reviewer` 에이전트로 diff 리뷰를 받는다. 특히 Hotkey/Inject/HUD 모듈과 권한 관련 코드는 리뷰 생략 금지.
3. 태스크의 완료 기준(구현계획서)을 확인하고, 수동 검증 절차를 `docs/VERIFY.md`에 추가한다.
4. 자동화 불가 항목(실제 마이크, 권한 다이얼로그, 터미널 주입)은 사용자에게 수동 확인을 요청하는 체크리스트를 출력한다 — 확인 없이 완료 선언 금지.

## 에이전트 위임 규칙

- macOS API 동작이 1%라도 불확실하면 추측하지 말고 `macos-api-researcher`에게 조사시킨다. (이 프로젝트 버그의 최대 원천은 API 추측이다.)
- 대규모 코드 탐색은 서브에이전트(Explore 포함)로 격리하고 결과 요약만 받는다.
- Phase 0 벤치마크 작업은 `bench-analyst`가 전담한다.
- 서브에이전트 정의를 수정하면 세션 재시작이 필요함을 사용자에게 알린다.

## 하지 말 것

- 시스템 설정/권한(TCC)을 코드로 우회하려는 시도.
- 클라우드 API 호출 추가 (완전 로컬은 제품 원칙 1 — 예외 없음).
- 문서 갱신 없는 "임시" 설계 이탈. 임시는 없다.
- 사용자 확인 없는 git push, 파일 삭제, 의존성 추가.
