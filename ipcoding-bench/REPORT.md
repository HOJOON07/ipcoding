## 2. 실험 A — STT

재현: `.venv/bin/python bench_stt.py` → `results_stt.csv` (조건: `-l ko`, no timestamps, RTF는 파일별 10회 반복·첫 회 워밍업 제외 중앙값).

| 모델 | 조건 | 평균 CER (A~D) | 용어 적중률 | RTF 중앙값 |
|---|---|---|---|---|
| large-v3-turbo q5 | initial_prompt 없음 | 0.2283 | 9/26 = 34.6% | 0.128 |
| **large-v3-turbo q5** | **initial_prompt 있음** | **0.1792** | **16/26 = 61.5%** | 0.128 |
| medium q5 | initial_prompt 없음 | 0.2674 | 2/26 = 7.7% | 0.128 |

**선정: `large-v3-turbo q5 + initial_prompt`(사전 용어 주입).**
근거: 판정 1순위인 용어 적중률에서 61.5% vs 34.6% vs 7.7%로 압도(동급 아님 — 부차 기준 불요). CER도 최저. initial_prompt 효과: 적중률 +26.9%p, CER −4.9%p. medium은 영어 용어를 거의 전부 한글 음차로 출력(useState→"유즈 스테이트" 류)해 탈락.

한계·특이사항:
- **RTF 측정 한계**: whisper-cli 서브프로세스 전체 벽시계라 호출마다 모델 로드가 포함됨 → 짧은 클립에서 로드 시간이 지배해 3조건 RTF가 동률(0.128). 앱은 모델 상주(TDD §3.3)라 실제 RTF는 이보다 낮을 것. 판정 3순위라 결론에 영향 없음.
- **초단문 취약**: s21 "커밋해줘"가 두 조건 모두 "거미 태조"로 전사(녹음 품질 문제 아님 — 2초 미만/무음 파일 0개 확인). 유형 D 커맨드형이 실사용 리스크. 시드 사전의 전문(全文) 치환("거미 태조→커밋해줘")으로 임시 방어 중.
- Apple SFSpeechRecognizer 기준선은 Swift 헬퍼 필요로 미측정 (README TODO).

## 3. 시드 사전 v0

**치환 후 용어 적중률: 26/26 = 100% → Phase 0 완료 기준(≥90%) 충족.** 치환 후 평균 CER(A~D): 0.1792 → 0.0891.

오염 위험 명시 (사용자 사전 확정 시 검토 필요):

## 4. 실험 B — LLM 교정·정돈

입력: 선정 STT(turbo+prompt) 실전사 30문장 + 시드 사전 치환 적용(실제 파이프라인 동일 지점). 조건: stream, think off, temperature 0.2, 워밍업 1회, 모델 전환 시 keep_alive=0. 채점: 보수 원칙("애매하면 실패"), 초안은 채점 스크립트에 근거와 함께 보존, 사용자 검토로 확정.

재현: `.venv/bin/python bench_llm.py --models qwen3.5:4b qwen3.5:9b --stt-model ggml-large-v3-turbo-q5_0.bin --stt-condition with_prompt [--prompt-file prompts/refine_v{1,2}.txt --out results_llm_v{1,2}.csv] [--no-prompt-dict(v2)]` · 채점: `score_llm_draft.py` / `score_llm_v1_draft.py` / `score_llm_v2_draft.py`.

### 4.2 선정: **qwen3.5:9b + 프롬프트 v2** (사용자 확정, 2026-07-08)

근거: BENCH.md §4 판정 순서 기계 적용 — 관문(의도 보존 실패 0건)을 v2 조건에서 9b만 통과. TTFT 1.58s ≤ 2s 충족. "동급이면 4B" 조항은 동일 조건(v2)에서 4b가 2건(s01·s04 문두 삭제 — v1·v2 연속 재발, 4b 고유 성향)으로 관문 미달이라 적용 불가.

**정직성 유의점**: 9b의 0건은 상당 부분 "원문 통과" 보수화로 달성됐다. 그 대가로 교정 성공률 22%(v0 78%에서 하락) — error log·Build·Generic·"만료→완료" 등 잔존 오인식을 원문 그대로 통과시킨다. 이 잔존분은 프롬프트가 아니라 **사전 확장으로 이관**하는 것이 방침이다(§7-④). 실제 파이프라인에서도 사전 치환이 담당하는 지점이고, 실험 A에서 사전 경로의 유효성(치환 후 100%)이 이미 입증됐다.

### 4.3 프롬프트 개선 이력 (실패 유형 → 조치 → 효과)

**v0 → v1** (`prompts/refine_v0.txt` → `refine_v1.txt`):

| v0 실패 유형 | v1 조치 | 효과 |

| 짧은 커맨드 날조 (9b "커밋해줘"→"commit message 작성 요청", "테스트 돌려"→"pytest 실행") | 규칙6 신설("고칠 것 없으면 그대로 출력") | s21 무변경 통과(양 모델). 단 9b s22에서 few-shot 예시 복사 역효과 1건 |
| 과잉 의역·부가 ("(300초)" 주석, 해줘→하시오) | 규칙2에 바꿔쓰기·부연·단위 환산 금지 | 해소. 부작용: 교정 소심화 시작 |
| 시스템 프롬프트 누출 (4b 2건: "AI 코딩 에이전트에게 전달") | 입력 구분자 `<<< >>>` 도입 + 지시문 문구 출력 금지 | 4b 누출 2→0. 부작용: 9b 구분자 에코 12건 |
| 어투 혼란 | 어투(반말/존댓말) 유지 명시 | 대체로 개선 |

**v1 → v2** (`refine_v1.txt` → `refine_v2.txt`):

| v1 실패 유형 | v2 조치 | 효과 |
|---|---|---|
| 4b 문두 맥락 삭제 ("useState 말고"·"이거 왜 안 되는지" 누락) | 군말 정의 축소(필러만, 주어·목적어·조건절·비교 대상 삭제 금지) + 맥락 보존 예시 | 부분 해소(4→2건). s01·s04 재발 — 4b 고유 성향, 프롬프트 한계효용 낮음 |
| 9b 구분자 에코 12건 | "구분자는 출력에 포함하지 않는다" 명시 | 12→1건 (잔존 1건은 후처리로 방어, §7-②) |
| 9b few-shot 예시 복사 (s22) | 예시를 벤치 문장과 안 겹치는 소재로 전면 교체 | 1→0 |
| 9b 사전 쌍 역적용 (재시도→"제시도", 린트→"링트에러고쳐줘") + 사전 문구 혼입 ("코믹 메시지가 아닌") | `{dictionary_pairs}`를 "(없음)"으로 비움 (입력 치환은 유지) | **역적용 2→0, 혼입 소멸 — 프롬프트 내 사전 주입이 원인임을 검증** |
| 교정 소심화 | 규칙1 완충("명백한 오표기는 사전 없이도 교정") | 실패 — 22%로 추가 하락. 사전 앵커 상실로 4b는 신규 훼손까지(rebase→"재베이스", force push→"포스 푸시"). 결론: 잔존 교정은 사전 확장으로 이관 |

## 5. 확정 스택 요약 + 예상 파이프라인 지연

**확정 스택**: whisper.cpp `large-v3-turbo q5` (language=ko, initial_prompt=사전 용어) → 사전 치환 (시드 v0 17항목) → llama.cpp `Qwen3.5 9B` (프롬프트 v2, 사전 주입 없음, think off, temp 0.2)

10초 발화 시나리오 추정 (측정 근거 병기):

| 구간 | 추정 | 근거 |
|---|---|---|
| T_raw (STT 완료) | ≈ 1.3s (상한) | RTF 0.128 × 10s. CLI 호출당 모델 로드 포함 상한치 — 상주 시 단축 예상 |
| T_first_token | ≈ 2.9s | T_raw + 9b TTFT p50 1.58s (사전 치환 ≈ 0) |
| T_ready (정돈 완료) | ≈ 3.2–3.7s | T_raw + 9b 총 생성 p50 1.90s. 벤치 평균 발화 7.1s 기준이라 10s 발화는 프롬프트 평가·출력 길이 증가로 상단쪽 추정 |

PRD §6.3 예산 대비: **L2 ≤ 4s — 충족 추정 (여유 0.3~0.8s로 크지 않음)**. L0 ≤ 1.5s — T_raw 상한 1.3s로 충족. L1 ≤ 2.5s — 현 설계는 교정·정돈 단일 호출(PRD §4)이라 L1 단독 경로는 별도 미측정, T_ready 기준으로는 2.5s 초과 → L1을 별도 축약 경로로 둘지는 Phase 2에서 재검토 필요.
**메모리 주의**: turbo q5 ≈ 0.6GB + 9b 6.8GB(Ollama 기준) ≈ **7.4GB로 PRD "9B급 합계 ≤ 7GB" 예산을 약 0.4GB 초과** 추정. llama.cpp 직접 통합 시 컨텍스트/KV 설정으로 조정 여지 — Phase 2에서 실측 후 PRD 예산 조정 또는 확정 필요.

## 7. 후속 권고

1. **TDD §3.6 수정안**: 사전은 ① 전사 직후 문자열 치환과 ② Whisper initial_prompt 주입만 유지하고, **LLM 프롬프트 내 용어 사전 주입은 제거**. 근거: v2 사전 제거 실험에서 9b 역적용 사고(재시도→제시도 등) 및 사전 문구 혼입이 소멸함을 검증.
2. **TDD §3.4 출력 정제에 구분자 스트립 추가**: `<<<`/`>>>` 에코가 v2에서도 1/60건 잔존 — 프롬프트만으로 완전 차단 불가, 후처리 방어선 필요.
3. **TDD §3.5 시스템 프롬프트를 v2로 교체** (`prompts/refine_v2.txt`).
4. **시드 사전 확장 후보** (실험 B 잔존 오인식 이관, 등록은 사용자 결정):
   - 안전: `error log→에러 로그`, `Build→빌드`(문두 한정 검토)
   - 오염 위험 검토 필요: `콘솔로 지워→콘솔 로그 지워`("콘솔로 이동" 오염), `검색기는→검색 기능은`(모호), `Generic→제네릭`(낮은 위험), ※ `Type→타입`은 "TypeScript" 파괴 위험으로 **등록 금지**
   - 불안정(조건마다 오류 상이 — 보류): `mid layer/미들 레이어→미들웨어`, `리페트링/리팩풀→리팩토링`
   - **s25 문맥 치환 후보 (사용자 결정 대기)**: `세션이 완료→세션이 만료` — 두 모델·세 프롬프트 모두 미교정한 잔존 오인식이나, "세션이 완료되면 로그아웃" 같은 정상 발화를 오염시키는 문맥 의존 치환이라 위험. 본인 말버릇 확인 후 결정할 것.
5. 초단문(D유형) 오인식(s21)은 사전 전문 치환으로만 방어 중 — Phase 1에서 짧은 발화 처리(예: 최소 녹음 길이, VAD 패딩) 검토 권장.
