# 입코딩 (Ip-Coding) — PRD v0.1

> **한 줄 정의**: 터미널에서 AI 에이전트와 대화하는 개발자를 위한 macOS 로컬 음성 입력 앱.
> 말은 두서없이 해도, 에이전트에게는 정돈된 프롬프트가 들어간다.

- 문서 버전: v0.2 (2026-07-07)
- 상태: 초안 — 미결정 항목은 §10 참조
- v0.2 변경: 3단계 모드 시스템 폐지 → 단일 파이프라인 + HUD 실시간 표시로 대체(§4), 핫키 ⌘+Fn 확정, Tab(원문 사용)/Esc(취소) 인터랙션 추가

---

## 1. 배경과 문제 정의

Claude Code, Codex 같은 AI 코딩 에이전트가 개발 워크플로우의 중심이 되면서, 개발자가 하루에 타이핑하는 텍스트의 상당 부분이 "코드"가 아니라 "에이전트에게 보내는 자연어 프롬프트"로 바뀌었다. 프롬프트는 길고, 문장형이며, 빈도가 높다. 이 입력을 음성으로 대체하면 손목 부담과 입력 속도 모두에서 이득이 크다.

기존 대안들의 한계:

| 대안 | 한계 |
|---|---|
| macOS 내장 받아쓰기 | 기술 용어·한영 혼용 인식 약함, 커스터마이징 불가 |
| Wispr Flow, superwhisper | 유료, 범용 받아쓰기 포지션, 터미널/코딩 도메인 최적화 없음 |
| Mac Whisper (오픈소스) | Apple SFSpeechRecognizer 기반이라 기술 용어에 약하고, 교정을 클라우드 LLM에 의존(텍스트 외부 유출), 블루투스 마이크 회피 설계 |
| ChatGPT 앱 등 앱 내장 음성 | 자기 앱 안에서만 동작, 터미널에 주입 불가 |

**핵심 관찰**: 범용 음성 받아쓰기는 이미 커모디티다. 승부처는 "터미널 AI 에이전트 코딩"이라는 도메인에 얼마나 깊이 들어가느냐다.

## 2. 타겟 사용자

- **주 사용자**: cmux/tmux 계열 터미널에서 Claude Code, Codex 등 AI 에이전트로 코딩하는 한국어 사용 개발자. 한국어 문장에 영어 기술 용어를 섞어 말한다("이 함수 async로 리팩토링하고 커밋해줘").
- **사용 환경**: Apple Silicon 맥. 에어팟/블루투스 헤드셋 착용 상태가 기본 시나리오(클램셸 모드, 카페 등 포함). 내장 마이크는 폴백.
- **프라이버시 민감도 높음**: 업무 코드/사내 정보가 발화에 섞이므로 음성·텍스트 모두 기기 밖으로 나가면 안 된다.

## 3. 제품 원칙

1. **완전 로컬**: 음성도, 전사 텍스트도, 교정 과정도 외부 서버로 나가지 않는다. (Mac Whisper와의 결정적 차별점 — 그쪽은 교정 텍스트가 클라우드 LLM으로 나간다)
2. **블루투스 퍼스트**: HFP 전환을 회피하지 않고 정면으로 지원한다. HFP의 16kHz는 Whisper 입력 규격과 일치하므로 인식 품질 손해가 거의 없다. PTT 방식이라 재생 음질 저하는 발화 중 몇 초로 국한된다.
3. **폴백 우선**: 어떤 처리 단계가 실패해도 원본 전사 텍스트는 반드시 주입된다. 사용자의 말이 증발하는 일은 없다.
4. **의도 보존**: 리라이팅은 사용자가 명시적으로 선택한 모드에서만, 주입 전 미리보기와 함께 동작한다.

## 4. 핵심 기능 — 단일 파이프라인 + 실시간 HUD

입코딩의 중심 설계. 모드 선택 없이 모든 발화가 동일한 파이프라인을 흐르고, 진행 과정 전체가 HUD에 실시간으로 보인다. 사용자의 선택권은 모드 토글이 아니라 **주입 직전의 3택 인터랙션**(기다림 / Tab / Esc)에 있다.

### HUD 상태 흐름

| 단계 | 트리거 | HUD 표시 | 목표 시점* |
|---|---|---|---|
| ① 녹음 | ⌘+Fn 다운 | 마이크 아이콘 + 라이브 파형 | 즉시 |
| ② 원문 | ⌘+Fn 릴리즈 | Whisper 전사 원문(사전 치환 적용) 즉시 표시 | ~1초 |
| ③ 다듬기 | 자동 | 원문이 흐려지고, 로컬 LLM이 교정+정돈한 문장이 토큰 스트리밍으로 타자 치듯 쓰여짐 | ~1–3초 |
| ④ 주입 대기 | 스트리밍 완료 | 완성 문장 + 힌트 표시: "잠시 후 자동 주입 · Esc 취소 · Tab 원문 사용" | 완료 후 N초 |
| ⑤ 주입 | 대기시간 경과 또는 사용자 선택 | 주입 후 HUD 사라짐 | — |

*10초 내외 발화, Apple Silicon 기준.

**설계 근거**: 원문이 1초 만에 먼저 보이므로 체감 대기시간이 짧고, 오인식 여부를 사용자가 즉시 판단할 수 있다. LLM 리라이팅은 원문을 제자리에서 고치는 게 아니라 새 문장을 처음부터 스트리밍 생성하므로, HUD는 "원문 → 새 문장이 흘러나옴" 형태로 표현한다.

### 주입 직전 3택 인터랙션

| 입력 | 동작 | 사용 상황 |
|---|---|---|
| (없음) | N초 후 다듬은 문장 자동 주입 | 기본. 대부분의 경우 |
| **Tab** | 다듬은 결과 대신 **②의 원문을 주입** | 이미 정확하게 말했고 리라이팅이 불필요/부적절할 때 |
| **Esc** | 아무것도 주입하지 않고 취소 | 잘못 녹음했거나 결과가 모두 틀렸을 때 |

N(자동 주입 대기시간)의 기본값은 프로토타입 자가 사용으로 확정한다(§10). 스트리밍이 진행 중일 때도 Esc는 항상 유효하다.

### LLM 처리 규칙 (교정 + 정돈, 단일 호출)

교정과 정돈은 별도 단계가 아니라 하나의 시스템 프롬프트로 한 번에 수행한다(2회 호출 대비 지연 절감).

- **교정**: 오인식·동음이의어·기술용어를 교체한다("유즈 스테이트" → `useState`, "깃 푸시" → `git push`). 확신이 없으면 원문 유지.
- **정돈**: 발화의 의도와 요구사항을 보존하면서 군말 제거, 순서 정리, 지시 명확화를 수행한다. 사용자가 말하지 않은 요구사항을 추가하지 않는다. 번역하지 않는다.
- 출력은 AI 에이전트 프롬프트로 바로 쓸 수 있는 평문(마크다운 구조화는 설정 옵션).
- LLM 타임아웃/오류 시 ②의 원문이 그대로 주입된다(제품 원칙 3).

## 5. 기능 명세

### 5.1 MVP (v1.0)

**입력**
- 글로벌 핫키 push-to-talk: **⌘+Fn**을 함께 누르는 동안 녹음, 릴리즈 시 처리 시작. 두 키 모두 modifier라 CGEventTap `flagsChanged`로 감지하며 일반 단축키와 충돌하지 않는다. 사용자 변경 가능.
- 입력 장치 선택: 시스템 기본 입력 따르기(기본값) / 특정 장치 고정. 블루투스 마이크 완전 지원.
- HUD: 화면 하단 미니 오버레이. §4의 5단계 상태(파형 → 원문 → 스트리밍 다듬기 → 주입 대기 → 소멸)를 표시. Tab/Esc 입력 수신.

**인식**
- whisper.cpp `large-v3-turbo` q5, Metal 가속, 앱 시작 시 메모리 상주.
- 언어 `ko` 고정(자동 감지 비활성) — 발화 단위 언어 감지 널뛰기로 인한 영어 문장 음차 사고 방지. 한영 혼용은 large 계열이 코드스위칭으로 처리.
- `initial_prompt`에 코딩 도메인 시드 어휘 주입 (사용자 사전과 연동).

**교정·정돈**
- llama.cpp 기반 로컬 LLM (후보: Qwen 계열 3–4B q4, 한국어+코드 이해 필수). 앱과 함께 자동 다운로드(첫 실행 시), 상주.
- 사용자 사전: 치환 규칙 CRUD UI. "들리는 대로 → 원하는 표기" 쌍. 사전은 L0 치환과 Whisper initial_prompt 양쪽에 반영.
- 실패 폴백: LLM 타임아웃(설정값, 기본 5초)/오류 시 L0 결과 주입.

**주입**
- 1차: 클립보드 + ⌘V 시뮬레이션 (주입 후 기존 클립보드 복원). 터미널 호환성 최고.
- 2차(설정): CGEventKeyboardSetUnicodeString 직접 주입 — 클립보드를 건드리기 싫은 사용자용.
- 대상 앱 제한 없음(포커스된 곳에 주입)이되, 최적화·테스트 우선순위는 cmux → iTerm2/Terminal/Ghostty → VS Code 통합 터미널.

**앱 셸**
- 메뉴바 상주 앱 (Dock 아이콘 없음). 메뉴: 모드 전환, 사전 편집, 설정, 일시정지.
- 온보딩: 권한 4종(마이크 / 손쉬운 사용 / 입력 모니터링) 안내 플로우 + 모델 다운로드 진행 표시.
- 설정: 핫키, 기본 모드, 입력 장치, 주입 방식, LLM 타임아웃, 모델 관리(재다운로드/삭제).

### 5.2 v1.x (MVP 직후)
- 히스토리: 최근 전사·교정 결과 로컬 저장, 재주입/복사. (오인식 패턴을 사전에 추가하는 진입점)
- 사전 자동 제안: 히스토리에서 사용자가 수정한 패턴을 감지해 치환 규칙 제안.
- 앱별 프로파일: 앱/터미널별 기본 모드 지정 (예: cmux는 L2, 슬랙은 L0).

### 5.3 v2 (탐색)
- 보이스 커맨드: "엔터", "취소", "지워" 등 발화를 텍스트가 아닌 키 입력으로 매핑.
- cmux 깊은 통합: pane 지정 주입, 에이전트 응답 상태 인지.
- 스트리밍 전사 미리보기 (말하는 도중 HUD에 중간 결과 표시 — 주입은 여전히 배치).
- 영어 UI/다국어 발화 모드 (해외 사용자 확장 시).

### 비범위 (Non-goals)
- Windows/Linux 지원 (v1 기준)
- 회의 녹음/전사, 자막 등 장시간 오디오
- 클라우드 API 교정 옵션 — 제품 원칙 1과 충돌하므로 넣지 않는다
- Mac App Store 배포 (샌드박스가 텍스트 주입과 양립 불가)

## 6. 기술 아키텍처

### 6.1 구성

```
┌─ 입코딩.app (Swift, 메뉴바) ─────────────────────────────┐
│  HotkeyManager      CGEventTap — PTT 감지                │
│  AudioCapture       AVAudioEngine → 16kHz mono PCM 버퍼  │
│  TranscribeEngine   whisper.cpp (SwiftPM, Metal) 상주    │
│  RefineEngine       llama.cpp (Metal) 상주, L1/L2 프롬프트│
│  Dictionary         치환 규칙 저장/적용 + initial_prompt  │
│  Injector           클립보드+⌘V / CGEvent 유니코드        │
│  HUD                NSPanel 오버레이 (상태·미리보기)      │
│  Settings/Onboard   SwiftUI                              │
└──────────────────────────────────────────────────────────┘
```

- 단일 프로세스로 시작. 추론 크래시 격리가 필요해지면 TranscribeEngine/RefineEngine을 XPC 헬퍼로 분리(구조는 프로토콜로 미리 추상화).
- 모델 파일은 `~/Library/Application Support/IpCoding/models/`에 저장, 앱 번들에 미포함(용량·업데이트 분리).

### 6.2 데이터 흐름

```
⌘+Fn 다운 → 녹음 시작 (HUD: 파형)
⌘+Fn 업   → 버퍼 마감 → Whisper 전사 → 사전 치환
          → HUD: 원문 즉시 표시 (이 시점의 텍스트 = Tab/폴백용으로 보존)
          → llama.cpp 교정+정돈, 토큰 스트리밍 → HUD에 실시간 렌더
             (타임아웃/오류 시 원문으로 폴백하고 ↓로 진행)
          → HUD: 주입 대기 (N초 / Tab=원문 / Esc=취소)
          → Injector 주입 → 클립보드 복원 → HUD 소멸
```

### 6.3 예산

| 항목 | 목표 |
|---|---|
| 상주 메모리 | Whisper turbo q5 ≈ 0.6GB + LLM: 4B급 채택 시 합계 ≤ 3.5GB / 9B급 채택 시 합계 ≤ 7GB (9B 채택은 곧 16GB 맥 전제) |
| 최소 사양 | Apple Silicon 16GB (8GB는 L0/L1 전용 모드로 동작 허용 여부 검증 필요) |
| 지연 (10초 발화) | L0 ≤ 1.5s / L1 ≤ 2.5s / L2 ≤ 4s |
| 유휴 CPU | ≈ 0% (핫키 대기만) |

### 6.4 주요 기술 리스크와 대응

| 리스크 | 대응 |
|---|---|
| 소형 LLM의 L2 정돈 품질 부족 | Phase 0 벤치마크에서 모델 3종 비교. 품질 미달 시 L2를 실험 기능으로 강등하고 L1 중심 출시 |
| 한영 혼용 오인식이 사전으로 커버 안 되는 경우 | initial_prompt 튜닝 + 히스토리 기반 사전 확장 루프 |
| 터미널별 붙여넣기 동작 차이 (bracketed paste 등) | 우선순위 터미널 4종 호환성 매트릭스 테스트 |
| macOS 버전별 권한/CGEvent 동작 변화 | 지원 최소 버전 확정(제안: macOS 14+) 후 CI에서 버전별 스모크 테스트 |
| HFP 전환으로 음악 끊김 체감 불만 | 발화 중에만 전환됨을 온보딩에서 안내 + 내장 마이크 강제 옵션 제공(선택지로만) |

## 7. UX 명세 요점

- **주입은 기본 자동**: 확인 클릭을 요구하면 타이핑보다 느려진다. 기본은 "N초 후 자동 주입"이고 Tab(원문)/Esc(취소)는 대기시간 동안의 거부권이다.
- **HUD는 포커스를 뺏지 않는다**: NSPanel non-activating — 주입 대상 앱의 포커스가 유지되어야 주입이 성립한다. Tab/Esc는 이벤트 탭 수준에서 가로채되(HUD 표시 중에만), 대상 앱으로 전달되지 않게 소비한다.
- **온보딩이 곧 첫인상**: 권한 3~4종 요구는 이 카테고리 앱의 최대 이탈 지점. 각 권한이 왜 필요한지 한 화면씩 설명 + 시스템 설정 딥링크.
- **사전 편집은 가볍게**: 메뉴바 → 사전 → 표에서 즉시 추가. 히스토리에서 "이 오인식 사전에 추가" 원클릭(v1.x).

## 8. 배포

- 채널: GitHub Releases (dmg) + Homebrew cask. 개발자 타겟이므로 Homebrew가 주력 유통.
- Developer ID 서명 + 공증 필수 (Apple Developer Program 연 $99).
- 자동 업데이트: Sparkle.
- 수익 모델: **미정** (§10). 오픈소스 여부가 리포 구성에 영향을 주므로 개발 착수 전 결정 권장.

## 9. 성공 지표 (초기)

- 활성 지표: 주간 발화 횟수 / 사용자 (개인 도구이므로 자기 자신이 첫 지표: 하루 50회 이상 자연스럽게 쓰는가)
- 품질 지표: 주입 후 사용자가 수동 수정한 비율 (히스토리 기반, 로컬 계산)
- L2 신뢰 지표: L2 미리보기에서 Esc 취소율
- 성능 지표: 릴리즈→주입 p90 지연

## 10. 미결정 사항 (Open Questions)

1. **수익/라이선스 모델** — 오픈소스 무료 / 유료 / 프리미엄. 리포 공개 여부와 직결.
2. **제품 영문명** — "입코딩" 로마자 표기(IpCoding? Ipcoding? 별도 영문명?). 번들 ID, 도메인 확보와 연결.
3. **자동 주입 대기시간 N** — 스트리밍 완료 후 몇 초 대기할지(후보: 0.5–1.5초), 그리고 "대기 없이 즉시 주입" 옵션 제공 여부. 프로토타입 자가 사용으로 결정.
4. **LLM 모델 최종 선정** — Phase 0 벤치마크로 결정. 후보 라인업(2026-07 조사): Qwen3.5 4B / Qwen3.5 9B / Kanana 1.5 8B / Gemma 4 E4B (전원 Apache 2.0, 상업 사용 가능) + 참고용 EXAONE 7.8B(연구용 라이선스라 출하 불가, 한국어 상한선 측정만). 비교 축: ① 4B vs 9B 크기 효용 ② 범용 최신 세대 vs 한국어 특화. 주의: Qwen3.5 Small은 멀티모달이라 llama.cpp 직접 통합 시 비전 파일(mmproj) 분리 이슈 검증 필요. 벤치마크는 Ollama(내부 엔진이 llama.cpp)로 수행하고 결과를 제품의 llama.cpp 직접 링크로 이전.
5. **최소 macOS 버전** — 14(Sonoma)+ 제안. macOS 26 신형 SpeechAnalyzer API 비교 벤치는 참고용으로만.
6. **8GB 맥 지원 범위** — L0/L1 전용 모드 허용 여부.
